Projects

Battleship ↡

This was the very first project I completed during my time at GA. The goal of the project was to take a game, and using JavaScript, HTML, and CSS, turn it into a computer app. This project incorporates event-driven programming, through use of DOM elements. I chose to do the game Battleship. I knew going into this project that there would be challenges of incorporating computer AI if I wanted the game to actually be fun. I enjoyed this challenge, along with figuring out how to get the user to place their ships, requiring each cell choice to be adjacent and in the same vertical/horizontal choice direction. If I had more time to work on this project, I would like to refactor a lot of my functions because they are quite large, and implement more smaller functions.

Plate Date ↡

This project is a Node/Express/MongoDB app similar to Yelp. It allows users to sign in using Google OAuth authentication, and search restaurants in their city using the Zomato third party API. Once a restauranted is selected and added, they can leave a review for their peers to view. Users can also check out other people's restaurants and reviews. This application implements full CRUD functionality. My favorite part of working on this app was incorporating the third party API, Zomato. I love data, and I think it was really interesting seeing how others design their database and why they chose to do it the way they did.A future goal I would like to implement in this application is to turn it into a dating app. I think there is something about food that brings people together, and people who eat together stay together!

A Perfect Evening ↡

This project was our final project in our GA software engineering immersive course, and the goal was to create a full stack app using the MERN stack: : MongoDB/Mongoose, Express, React, NodeJS. Token based authentication, and authorization is also used. The app I chose to create is a cheese and wine tracker. It comes with a cheese and wine pairing guide for reference, to help users get some idea of what they should be eating with the type of wine they choose. The user can search wines from Global Wines database, and select a wine to add to their 'wine diary'. In their diary, they can add cheeses that they paired with the wine, and add additional notes so they can remember for next time. One challenge I had while making this app was finding a wine API that had the type of wine (merlot, cabernet..). However, I was unable to find one for free. I wanted to be able to offer a cheese recommendation based on the type of wine, so a future goal is to have the user select what type of wine it is, and work with that. My favorite part of this app along with others I have worked with is working with the third party API. I really enjoy seeing how others choose to store their data adn design their models.
